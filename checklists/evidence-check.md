# Evidence Strength Checklist

ใช้ตรวจสอบก่อน publish — ทุกบทต้องผ่าน

## Claims Audit

สำหรับทุกข้อความสำคัญในบท:

| ข้อความ | Source | Level | Verify First? |
|---------|--------|-------|---------------|
| (copy claim) | (file/docs/URL) | `[✓]` `[→]` `[?]` `[!]` | Yes / No |

## ตรวจสอบทีละประเภท

### Direct `[✓]` — ต้องมี source
- [ ] ระบุไฟล์หรือ URL ได้
- [ ] ดึงข้อมูลจริง ไม่ใช่จากความจำ
- [ ] Stars/language/version เป็นปัจจุบัน

### Inferred `[→]` — ต้องอธิบายเหตุผล
- [ ] อธิบายว่าอนุมานจากหลักฐานอะไร
- [ ] ไม่อนุมานข้ามขั้น (เช่น เห็น SQLite → อนุมานว่าใช้ ORM)

### Assumed `[?]` — ต้องระบุว่าเป็นสมมติฐาน
- [ ] กำกับชัดเจนว่าเป็นการสมมติ
- [ ] อธิบายว่าทำไมต้องสมมติ (ไม่มีข้อมูล / นอก scope)
- [ ] ไม่ใช้สมมติฐานโดยไม่จำเป็น

### Unverified `[!]` — ต้องมีแผนกลับมาตรวจ
- [ ] ระบุสิ่งที่ต้องตรวจสอบ
- [ ] ระบุความเสี่ยงถ้าไม่ตรวจสอบ
- [ ] ไม่ใช้ `[!]` ในข้อความสำคัญที่เปลี่ยน architecture ได้

## Final Check
- [ ] ไม่มีข้อความสำคัญที่เป็น `[!]`
- [ ] `[→]` ทุกอันมีเหตุผลอธิบาย
- [ ] `[?]` ทุกอันจำเป็นจริงๆ
- [ ] `[✓]` ทุกอันมี source
