# Textbook Quality Checklist

ใช้ตรวจสอบบทก่อน publish — Standard Lesson ขึ้นไป

## Teaching Format (9 ส่วน)

- [ ] 1. เป้าหมายบทเรียน — ชัดเจน วัดได้
- [ ] 2. ภาพรวมแนวคิด — 3-5 บรรทัด อ่านแล้วรู้ว่าจะเรียนอะไร
- [ ] 3. ทำไมถึงออกแบบแบบนี้ — มี tradeoffs, constraints
- [ ] 4. ไฟล์จริงที่เกี่ยวข้อง — พร้อมคำอธิบาย
- [ ] 5. อธิบาย flow จากบนลงล่าง — step-by-step
- [ ] 6. จุดที่มือใหม่มักเข้าใจผิด — อย่างน้อย 2 จุด
- [ ] 7. แบบฝึกหัดท้ายบท — 3-5 ข้อ (ทบทวน, วิเคราะห์, ลงมือทำ)
- [ ] 8. คำถามชวนคิด — เปิดให้คิดต่อ
- [ ] 9. สิ่งที่ควรอ่านต่อ — มีลิงก์จริง

## Learning Path

- [ ] บอก prerequisites
- [ ] บอก learning outcomes
- [ ] บอก starting files สำหรับทดลอง

## Code Reading Pipeline

- [ ] ผ่าน 7 ขั้น — ทุกขั้นมี `[✓]` หรือ `[!]`
- [ ] ไม่ได้เขียน architecture chapter จาก README อย่างเดียว

## ภาษา

- [ ] ไทยทั้งหมด (ยกเว้นชื่อ, code, URL)
- [ ] อ่านแล้วเหมือนอาจารย์สอน — ไม่ใช่แค่ข้อมูล
- [ ] ไม่มีศัพท์เทคนิคที่ไม่อธิบาย

## Exercise Quality

- [ ] ข้อ 1: ทบทวนความเข้าใจ (ตอบได้จากเนื้อหา)
- [ ] ข้อ 2: วิเคราะห์ (ต้องคิดเอง)
- [ ] ข้อ 3: ลงมือทำ (ต้องเขียนโค้ด)
- [ ] (optional) ข้อ 4: Challenge
- [ ] มีเฉลย

## Final Check

- [ ] อ่านทั้งบทตั้งแต่ต้นจนจบ — flow ลื่นไหล?
- [ ] ปิด Obsidian เปิดใหม่ — links ยังใช้ได้?
- [ ] ให้คนอื่นอ่าน — เข้าใจไหม?
