# 📖 Glossary Template

> รวบรวมจากทุกบท — คำศัพท์ที่ปรากฏครั้งแรก + อธิบายแบบคนไทยเข้าใจ
>
> **2 รูปแบบ:** Standard (คำทั่วไป) และ Deep (core terms — ตาม Deep Terminology Protocol)

---

## Standard Format — สำหรับคำทั่วไป

```
### <คำศัพท์>
- **ความหมาย:** (อธิบายด้วยภาษาไทยง่ายๆ — เหมือนอธิบายให้เพื่อนฟัง)
- **ศัพท์เทคนิค:** (คำภาษาอังกฤษ — ถ้ามี)
- **ปรากฏใน:** [[บทที่ X]]
- **ไฟล์ใน repo:** `path/to/file`
- **เกี่ยวข้องกับ:** (ลิงก์ไปคำอื่นใน glossary)
```

---

## Deep Format — สำหรับ core terms (ใช้ Deep Terminology Protocol)

> ใช้เมื่อคำนี้ปรากฏ 3+ ครั้งใน repo หรืออยู่ใน `glossary_candidates` จาก book-plan

```
### <Term> (ครั้งแรกที่เจอ — [[บทที่ X]])
├── คำจำกัดความ — อธิบายเป็นภาษาไทยง่ายๆ 1-2 ประโยค
├── มันทำงานยังไง — กลไก, flow, diagram ถ้าจำเป็น
├── ทำไมถึงสำคัญ — impact ต่อระบบ, ถ้าไม่มีจะเป็นยังไง
└── ตัวอย่างใน repo [✓] — `path/to/file.ts → function() @ <sha>`

**เกี่ยวข้องกับ:** [[Term A]], [[Term B]]
```

## ตัวอย่างจริง

### Standard

```
### Graph
- **ความหมาย:** โครงสร้างข้อมูลที่แทนความสัมพันธ์ — 
  เหมือน Facebook friends แต่ละ node คือคน แต่ละ edge คือ "เป็นเพื่อนกัน"
- **ศัพท์เทคนิค:** Graph
- **ปรากฏใน:** [[บทที่ 3]]
- **ไฟล์ใน repo:** `packages/core/graph/`
- **เกี่ยวข้องกับ:** [[Node]], [[Edge]]
```

### Deep

```
### Plugin System (ครั้งแรกที่เจอ — [[บทที่ 3]])
├── คำจำกัดความ: ระบบที่ให้ผู้ใช้เพิ่มความสามารถใหม่ให้โปรแกรมโดยไม่ต้องแก้โค้ดหลัก — 
│   เหมือนปลั๊กไฟที่ให้คุณเสียบเครื่องใช้ไฟฟ้าเพิ่มได้เรื่อยๆ
├── มันทำงานยังไง: โปรแกรมหลักประกาศ "จุดต่อ" (hook points) → plugin ลงทะเบียนตัวเองที่จุดนั้น → 
│   เมื่อถึงจังหวะที่กำหนด โปรแกรมหลักจะเรียกทุก plugin ที่ลงทะเบียนไว้
├── ทำไมถึงสำคัญ: ถ้าไม่มี plugin system ผู้ใช้ต้อง fork โปรเจคแล้วแก้โค้ดเอง → 
│   อัปเดตไม่ได้, ขัดแย้งกับคนอื่น, ecosystem ไม่เกิด
└── ตัวอย่างใน repo [✓]: `packages/core/plugins/loader.ts → loadPlugins()` — 
│   อ่าน config → resolve path → dynamic import → register hooks @ abc1234

**เกี่ยวข้องกับ:** [[Hook System]], [[Module Loader]]
```

## กฎการเลือกรูปแบบ

| เงื่อนไข | รูปแบบ |
|---------|--------|
| core term (3+ ครั้งใน repo หรืออยู่ใน `glossary_candidates`) | Deep |
| คำทั่วไปที่รู้กันอยู่แล้ว (API, HTTP, etc.) | Standard |
| คำที่เจอครั้งเดียว — ไม่ใช่ core | Standard |
| คำที่ยังไม่เห็นใน repo → `[!]` | Standard (รออัปเดตเป็น Deep เมื่อมีหลักฐาน) |
