> 🧭 **Guiding Questions** — ก่อนเดินอ่านโค้ด ให้ถามตัวเอง:
> 1. ไฟล์นี้แก้ปัญหาอะไร? (1 ประโยค — ไม่ใช่ "จัดการ session" แต่เป็น "แก้ race condition ตอนสร้าง session พร้อมกัน")
> 2. จุดไหนในไฟล์ที่ใช้ concept สำคัญจากบทก่อนหน้า? (เชื่อมกับ learning path)
> 3. มีบรรทัดไหนที่อ่านครั้งแรกแล้ว "หืม?" — นั่นคือจุดที่ต้อง Deep Dive
> 4. ฟังก์ชันนี้ใช้ pattern อะไร? (factory, observer, strategy...) — บอกชื่อ pattern
> 5. ถ้าจะ refactor ไฟล์นี้ให้ดีขึ้น — จะเปลี่ยนอะไร? (ดูจาก comments/TODO/issues)
